# 工程宪法 · Constitution

> 本文件是本项目 **spec-driven 流程的最高原则来源**。所有 `spec-*` skill 执行前读取它并以它为准；各 skill 的「核心原则」只是这里的精简内置默认。
> **流程细节**（七阶段、各阶段完成标准、目录与命名约定）见 `docs/engineering/workflow.md`；本宪法只管**不可妥协的原则**。
> 冲突时优先级：**本宪法 > `workflow.md` > skill 内置默认**。
> 修订：通过 PR/评审更新，并提升 `version`。

## 原则（不可妥协）

1. **单一事实源** —— 同一条信息只在一处维护，改一处即全链生效；产出物之间靠引用而非复制。
2. **先文档、后代码** —— 任何改动先回流 PRD（必要时原型、plan），最后才动代码；需求范围变化必须先更新文档。
3. **续编不重排** —— `R/F`、`AE/AC`、实现单元 `U` 一律往后续编，**绝不重排已有编号**，否则历史追溯与测试引用全部失效。
4. **双向可追溯 + 覆盖矩阵** —— 可追溯链 `R/F`（要做什么）→ `U`（怎么做）→ `AE/AC`（怎么算做对了）；每个特性维护一张覆盖矩阵，逐行可核对，无漏验收、无悬空、无漏实现。
5. **显式非目标** —— 每份 PRD 写清「这次不做什么」，主动防范围蔓延。
6. **非功能约束显式** —— 性能/并发/安全权限/可用性/兼容性要写；确无要求也显式写「无特殊要求」，不靠默认。
7. **右尺寸** —— 简单清晰的需求少追问、直接定稿；模糊或跨切的才多轮澄清。澄清一次一问。
8. **可测句式（高风险）** —— 复杂/高风险需求用「当 <触发> 时，系统应 <可观测行为>」类结构化句式书写，消除歧义。
9. **阶段交接** —— 每个阶段产出后明确指向下一步；已有特性的增量改动走 `/spec-change`，不另起炉灶。

## 工程基线（编码期标准）

> 上节「原则」管设计/文档期的「为什么」；本节是落到**编码期**的强制基线。spec-* 在 plan 之后把实现交给 `ce-work` / `ce-code-review` / `verify`，下列标准靠本节 + 项目 `CLAUDE.md` 注入生效（按项目技术栈细化）。

1. **安全合规前置** —— 严禁硬编码密钥/账号/内网地址；敏感字段（手机号/身份证/地址/交易）默认脱敏，禁明文返回与日志；严禁高危操作（清空全表、关权限校验、绕过登录）；接口默认带权限校验、防越权/防重放兜底。
2. **禁止幻觉** —— 不编造不存在的接口/字段/方法/配置/业务规则；不确定处主动标「待确认」。
3. **零魔法值** —— 固定状态/类型/阈值抽枚举或常量，代码中不硬编码。
4. **强健壮性** —— 入参、外部返回、集合做非空/边界/合法性校验；资源（流/连接/线程）用完必释放。
5. **异常分层兜底** —— 全链路捕获，区分业务/系统/第三方异常，友好提示，不裸抛原生堆栈。
6. **最小改动** —— 不随意重构、不批量改无关代码、不破坏既有正常逻辑。
7. **性能基线** —— 禁循环嵌套查询/循环单条 DB 操作；批量走批量；查询走索引/分页/条件过滤，杜绝全表扫描。
8. **可追溯提交** —— 小步增量、单一功能单一提交；合入主干前 squash 成干净可回滚的正式提交。
9. **字段长度全链路一致** —— `varchar(N)` 的 N 是字符数（utf8mb4 下为 N 个汉字），按业务最大汉字数定义、不按字节估算；**DB 字符数为唯一真值**，后端入参按字符数前置校验（不靠 DB 截断兜底）、前端 `maxLength` 与接口文档都对齐该字符数；DB / 后端 / 前端三处一次同步到位。

## 编号与术语（规范定义）

| 缩写 | 含义 | 续编规则 |
| --- | --- | --- |
| **R / F** | 功能需求编号（一份 PRD 内二选一） | 往后续编，不重排 |
| **AE / AC** | 验收标准编号，须标注覆盖的 R/F | 往后续编，不重排 |
| **U** | plan 里的实现单元编号 | 往后续编，不重排 |
| **NNN** | 特性序号，PRD/原型/plan/迁移**同号串联** | 新特性 = 现有最大号 + 1 |

详细写法与示例见 `/spec-prd` 术语表。

## skill 与宪法的关系

`/spec-prd`（需求）· `/spec-prototype`（原型）· `/spec-design`（技术设计）· `/spec-plan`（任务计划）· `/spec-change`（变更回流）· `/spec-check`（覆盖矩阵体检）—— 全部以本宪法为**原则基线**、以 `docs/engineering/workflow.md` 为**流程基线**。
