# <NNN> <特性名> · PRD

## 1. 背景与问题
〔为什么做、当前痛点、不做会怎样〕

## 2. 目标 / 非目标
- **目标**：〔本次要达成的结果〕
- **非目标**：〔这次明确不做的，防范围蔓延〕

## 3. 用户与角色
〔涉及哪些角色、各自诉求与权限差异〕

## 4. 关键决策
〔选了什么、为什么、放弃了哪些备选；澄清得到的结论记这里〕

## 5. 功能需求（R/F，续编不重排）
- **R1** 作为〔角色〕，我想〔做什么〕，以便〔价值〕。
- **R2** …

> 复杂/高风险条目用可测句式消歧：
> - 当 〔触发〕 时，系统应 〔可观测行为〕。
> - 若 〔条件〕，则系统应 〔可观测行为〕。
> - 在 〔某状态〕 期间，系统应 〔可观测行为〕。

## 6. 非功能约束
〔性能 / 并发 / 安全与权限 / 可用性 / 兼容性；确无要求就写「无特殊要求」，不要留空〕

## 7. 验收标准（AE/AC，每条标注覆盖的需求号）
- **AE1（R1）** 〔可观测、可判定的验收场景〕
- **AE2（R8）** …

## 8. 覆盖矩阵
| 需求 | 覆盖它的验收 | 非功能约束 |
| --- | --- | --- |
| R1 | AE1 | 无 |
| R8 | AE2、AE5 | 约满判定需并发安全 |

> 规则：每条 R/F 至少一条 AE/AC（空 = 漏验收）；每条 AE/AC 至少指向一条存在的 R/F（悬空 = 多余验收或漏写需求）。

## 9. 成功指标
〔怎么衡量做成了，尽量可量化〕

## 10. 依赖与假设
〔外部依赖、前置假设〕

## 11. 待解决问题
〔未决项；澄清中未拍板的记这里〕
