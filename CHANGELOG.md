# Changelog

本套件的演进记录。格式参考 [Keep a Changelog](https://keepachangelog.com/)。

## 设计阶段独立 + 企业规约对齐（2026-06）

把套件从「**5 个 skill（设计揉在计划里）**」演进为「**6 个 skill + `spec-init` 脚手架**」，流水线由 `prd→prototype→plan` 变为 **`prd→prototype→design→plan`**（8 阶段），并对齐企业级开发规约（参阿里巴巴《Java 开发手册》/前端规约）。共 16 个 PR（#1–#16）。

### 新增

- **`/spec-design` 独立成阶段**——概要设计 + 数据 ER + 详细设计，产出 `docs/engineering/design/`；流水线重排为 8 阶段。(#1)
- **设计四要素**：前端设计衔接、接口契约、并发与幂等、权限与可见域。(#2)
- **设计再补全**：核心业务流程图、权限四级（菜单/按钮/接口/数据）、可观测与审计、代码结构落点、接口请求-返回示例。(#3)
- **`spec-init` 脚手架资源包**：`constitution.md` / `workflow.md` / `CLAUDE.md` 模板 + `init-project` 脚本，新项目一键初始化；独立于 6 个 skill。(#4, #15)
- **`workflow.md` 模板**（8 阶段，技术栈占位）。(#4)
- **`CLAUDE.md` 编码规约模板**：命名/注释（参阿里 Java 手册，#10）、前端规范 11 节（参阿里前端规约 + Vue/React 官方 Style Guide，#14）、后端规范（OOP/集合/并发/控制语句/事务/异常/日志，参阿里 Java 手册，#16）。
- **constitution「工程基线」**：编码期 11 条（安全合规、零魔法值、命名语义化、注释即文档、字段长度全链路一致…）。(#3, #10)
- **全链路产出样例** `examples/`（会议室预约：PRD→设计→计划→迁移，含阅读导引）。(#8)

### 规约

- **数据**：回滚/down 段必填、时间戳统一、**字段长度全链路一致**（业务字符数为唯一真值，前后端按字符对齐，#3）；覆盖异构/国产库（SQL Server / Oracle / 达梦 / 人大金仓 / 神州通用，#11）；**生成 SQL 前先确定库语义，未记录则询问用户**（#12）。
- **设计**：**右尺寸总纲**（详细设计只锁 AI 推不出/推错代价高的决策，#6）；**概要 + ER 确认检查点** + 「一份文档分步走」+ 依赖标注（#7）。
- **spec-check 校验扩展**：新增 C11–C15（设计落地、回滚段、时间戳、字段长度、设计章节完整）；并明确**命名/注释/前后端编码规约属编码期，归 `/code-review`**，不在 spec-check 范围。

### 文档 / 修复

- README：「右尺寸」独立解释（#13）、「为什么设计是一份文档」（#7）、阶段号 `④–⑦`→`⑤–⑧` 修复（#9）。
- `init-project` 命令加 `-ExecutionPolicy Bypass` 临时授权（规避 Windows Restricted，#5）。

### 重构

- 项目脚手架从 `spec-prd/templates/` 移到独立的 `spec-init/`（`spec-prd/templates/` 只留产出模板 `prd.md`）；`install` 改为「先删后 copy」，避免重构后用户级残留旧文件。(#15)
